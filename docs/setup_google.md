# Google API セットアップガイド

`calendit` で Google カレンダーを操作するには、Google Cloud Console で API を有効にし、認証情報（クライアント ID とクライアント シークレット）を取得する必要があります。

## 手順 1: プロジェクトの作成と API の有効化

1.  [Google Cloud Console](https://console.cloud.google.com/) にアクセスします。
2.  上部のプロジェクト選択メニューから「新しいプロジェクト」を作成します（例: `calendit-project`）。
3.  左サイドバーの「API とサービス」 > 「ライブラリ」を選択します。
4.  検索窓に **"Google Calendar API"** と入力し、選択して「有効にする」をクリックします。

## 手順 2: OAuth 同意画面の設定

1.  「API とサービス」 > 「OAuth 同意画面」を選択します。
2.  User Type で「外部」を選択し「作成」をクリックします（個人の場合はこれでOK）。
3.  アプリ名（例: `calendit`）とユーザーサポートメールを入力します。
4.  「保存して次へ」を最後まで進め、ダッシュボードに戻ります。
5.  **重要**: 公開ステータスを「テスト」のままにする場合は、「テストユーザー」に自分の Google アカウントを追加してください。

## 手順 3: 認証情報の作成と JSON のダウンロード

1.  「API とサービス」 > 「認証情報」を選択します。
2.  「+ 認証情報を作成」 > 「OAuth クライアント ID」を選択します。
3.  アプリケーションの種類で **「デスクトップ アプリ」** を選択し、名前を入力して「作成」をクリックします。
4.  表示されたダイアログで「JSON をダウンロード」をクリックします。
5.  ダウンロードしたファイルを `credentials.json` などの名前で保存します。

## 手順 4: `calendit` への取り込み

ターミナルで以下のコマンドを実行して、ダウンロードした JSON ファイルを `calendit` に読み込ませます。

```bash
# JSONファイルを指定して設定
calendit config set-google --file ~/Downloads/credentials.json
```

実行後、セキュリティのために元の JSON ファイルを削除するか確認されます。`y` を入力すると削除されます。

## 手順 5: ログイン

設定が完了したら、以下のコマンドで Google アカウントにログインします。

```bash
calendit auth login google
```

ブラウザが開くので、許可を与えてください。完了メッセージが表示されたらターミナルに戻ります。

---
これでセットアップは完了です！ `calendit cal list` でカレンダーが表示されるか確認してみましょう。
