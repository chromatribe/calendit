# 🔰 calendit 初心者ガイド（非エンジニア向け）

**「ターミナル（黒い画面）を使うのは初めて」「コマンドのことはよく分からない」** という方でも大丈夫です！  
このガイドでは、プログラミングの知識がなくても `calendit` を使い始められるように、**最小限の手順**だけをまとめています。

画面に表示されるコマンドをコピー＆ペーストしていくだけで、カレンダーの予定をターミナルで操れるようになりますよ！

---

## 📌 知っておくべき３つの言葉

まずはこれだけ覚えておきましょう！

1. **ターミナル**: 文字を打ち込んでパソコンに命令を出す「黒い画面」のことです。（macOS なら「ターミナル.app」を使います）
2. **コマンド**: ターミナルに入力する「1行の命令」のことです。コピーして貼り付け、Enterキーを押すだけで実行されます。
3. **コンテキスト**: calendit の中で使う「カレンダーのあだ名」です。例えば、仕事用のGoogleカレンダーに `work` というコンテキスト（名前）を付ければ、今後は `work` と指定するだけでそのカレンダーを操作できるようになります。

---

## 🚀 ステップ1：インストールしよう

calendit を動かすためには、**Node.js（ノードジェイエス）** という土台が必要です。

### 1. Node.js が入っているか確認する
ターミナルを開き、以下の1行をコピーして貼り付け、Enterキーを押してください。

```bash
node --version
```

- **`v18.0.0` 以上の数字（例: `v20.11.0` など）が出た場合**: おめでとうございます！次に進んでください。
- **「見つかりません」や `v17` 以下が出た場合**: 公式サイト（[https://nodejs.org/](https://nodejs.org/)）から **LTS（推奨版）** をダウンロードしてインストールしてください。その後、**ターミナルを一度閉じて、開き直してから** 次へ進んでください。

### 2. calendit をインストールする
Node.js が準備できたら、次のコマンドをターミナルに貼り付けて実行します。

```bash
npm install -g calendit
```

> [!TIP]
> 成功したか確認するために、以下のコマンドを実行してみましょう。
> ```bash
> calendit --version
> ```
> バージョン（数字）が表示されれば、インストールの完了です！🎉

## 📅 ステップ2：カレンダーを登録しよう（初期設定）

次に、「どのカレンダーを使うか」を設定します。
使いたいサービスを以下の３つから選んでください。

### 🅰️ Googleカレンダーを使う場合

> [!NOTE]
> Googleのクラウド設定など、少し細かい手順が必要です。  
> 詳しい画面操作の解説は **[Google設定ガイド](setup_google.md)** を見ながら進めてください。

**おおまかな流れ:**
1. Google Cloud でキー（JSONファイル）を発行し、パソコンにダウンロードします。
2. 以下のコマンドで、そのキーを calendit に教えます。
   ```bash
   calendit config set-google --file "ダウンロードしたJSONファイルのパス"
   ```
3. 「コンテキスト（あだ名）」をつけます（ここでは `my-google` とします）。
   ```bash
   calendit config set-context my-google --service google --calendar primary --account "あなたの@gmail.com"
   ```
4. ブラウザが開くので、ログインして連携を許可します。
   ```bash
   calendit auth login google --set my-google
   ```

### 🅱️ Outlook（Microsoft）カレンダーを使う場合

> [!NOTE]
> Microsoft Azure（Entra）での設定が必要です。  
> 詳しい画面操作の解説は **[Outlook設定ガイド](setup_outlook.md)** を見ながら進めてください。

**おおまかな流れ:**
1. Microsoft 側でアプリを登録し、「クライアントID」をコピーします。
2. 以下のコマンドで、IDを calendit に教えます。
   ```bash
   calendit config set-outlook --id "コピーしたクライアントID"
   ```
3. コンテキストをつけます（ここでは `my-outlook` とします）。
   ```bash
   calendit config set-context my-outlook --service outlook --calendar primary --account "あなたの@outlook.com"
   ```
4. ブラウザが開くので、ログインして連携を許可します。
   ```bash
   calendit auth login outlook --set my-outlook
   ```

### 🍏 このMacのカレンダーだけを使う場合（macOS限定）

> [!TIP]
> クラウド設定は不要です！Macに入っている「カレンダー.app」の予定をそのまま使います。

1. ブリッジ用アプリを起動します。
   ```bash
   calendit macos bridge start
   ```
   > ※初回は画面に「カレンダーへのアクセスを求めています」と表示されることがあります。その場合は**必ず「許可」**してください。

2. カレンダーを選択し、コンテキスト名を付ける対話式ウィザードを進めます。
   ```bash
   calendit macos setup
   ```
   画面の案内に従って、使いたいカレンダーと好きな名前（コンテキスト）を入力してください。

## 🔍 ステップ3：使ってみよう！

設定お疲れ様でした！さっそく今日の予定を見てみましょう。
（※ `--set` の後ろには、ステップ2で自分が付けたコンテキスト名を入れてください）

```bash
calendit query --set my-google --start today --format md
```

ターミナルに今日の予定が表示されましたか？
予定が0件だったとしても、エラーの赤い文字が出なければ**成功**です！✨
