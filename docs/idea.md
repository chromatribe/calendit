## 実装の詳細メモ

### コンテキスト別認証 (2026-04-12実装)
- `accountId` をトークンファイル名に含めることで、複数の Google アカウントを同一マシン上で使い分け可能にした。
- `getServiceForContext` において、コンテキスト設定から適切なトークンを自動ロードする仕組み。

### テスト自動化 (2026-04-12実装)
- `src/test_runner.ts` が `npm run dev` 経由で実際の CLI を叩き、出力を検証する。
- タイムゾーンのオフセット（+09:00）が含まれているか等の、実地に近い検証が可能。

### アーキテクチャ更新 (2026-04-12実装)
- `AbstractCalendarService` を基底クラスとして導入。
- 各サービス固有の「つまずき（API仕様差分）」を `handleError` や `normalize` フックで吸収。
- `ProviderCapabilities` により、Web会議連携などの機能可否をコア側から判別可能にする。

### プロフェッショナル品質管理 (2026-04-12実装予定)
- `docs/tests.md` の機械読み取りに対応。
- 限界文字数、特殊記号、異常な認証状態、未来/過去の極端な日付などの「意地悪な」テストケースを実装。
- LLM オートメーションを意識した初期設定（オンボーディング）の自動検証。
