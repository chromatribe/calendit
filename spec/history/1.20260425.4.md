# 1.20260425.4

## `apply` の取得範囲と Google 時刻帯

- **`calendit apply`**: オプション `--start` / `--end` を追加（`query` と同じ日付解釈）。入力ファイルの自動日付範囲と和集合（端点 min/max）を `listEvents` に用いる。`--end` だけの指定は不可。ログに有効な同期区間 `Sync range` を出す。既存 ID が取れないまま新規扱いになると **WARN**（`--start` / `--end` 案内付き）。実装: [`src/core/timeRangeForQuery.ts`](../../src/core/timeRangeForQuery.ts)・[`src/commands/apply.ts`](../../src/commands/apply.ts)・[`src/core/applier.ts`](../../src/core/applier.ts)。
- **リファクタ**: `query` の期間パースを [`src/core/timeRangeForQuery.ts`](../../src/core/timeRangeForQuery.ts) へ集約、[`src/commands/query.ts`](../../src/commands/query.ts) はそれを利用。
- **Google Calendar API**: [`src/services/google.ts`](../../src/services/google.ts) の `create` / `update` で `start`・`end` の `dateTime` に IANA 名の `timeZone`（`Intl` のローカル帯域）を付与。Outlook 実装の方針に近づく。
