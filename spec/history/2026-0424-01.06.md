# 2026-0424-01.06

## npm レジストリ向けのパッジングとバージョン形式

- **`package.json#version`**: [semver](https://semver.org/) 準拠（例: `2026.4.24`）。`2026-0424-01.05` 形式は `npm publish` では `Invalid version` になるため、公開可能な 3 セグメント形式に変更。
- **`files`**: `bin/`、`dist/`、ルート `README.md` / `LICENSE` のみを tarball に同梱。`native/**/.build` 等の巨大物は同梱しない（旧 `.npmignore` だけに依存すると 50MB+ になっていた）。
- **ライフサイクル**: `prepare`（post-install での常時 `build`）を廃止。`prepack` で `npm test`（`npm pack` / `npm publish` 前に完走）。clone 直後の開発者は `npm run build` が必要（CONTRIBUTING に明記）。
- **テスト**: `docs/tests.md` TC-INST-01 の期待を `2026-` から `2026` へ（semver 表記 `2026.4.24` に合わせる）。
