# バージョン 2026-0412-01.13

## 概要
テスト品質の徹底的な強化。ダイナミックなテストランナーの導入と、過酷な使用環境を想定したエッジケーステストの拡充。

## 変更・追加内容
### テストインフラの刷新
- **ダイナミックテストランナー**: `src/test_runner.ts` を刷新。`docs/tests.md` から直接テスト定義を読み取って実行する仕組みに変更。
- **機械読み取り形式の定義**: Markdown 内に `sh` ブロック（実行コマンド）と `expect` / `expect-fail` ブロック（期待結果）を記述することで、ドキュメントと実行ロジックを完全に同期。

### テストケースの拡充 (意地悪なテスト)
- **TC-14: 限界文字数テスト**: 2000文字を超えるサマリや場所の入力に対する耐性を検証。
- **TC-15: 破損ファイル耐性**: 無効な形式の行が混在する Markdown ファイルの処理継続性を検証。
- **TC-16: Unicode ストレス**: 絵文字や特殊記号を含む予定の同期を検証。
- **TC-19: 認証情報破損テスト**: トークンファイルが不正な状態での適切なエラー誘導を検証。
- **TC-20: 時刻矛盾バリデーション**: 未来開始・過去終了の予定に対するツール側での検証を強化。

### ロジックの堅牢化 (Hardening)
- **パースロジックの改善**: `Formatter.fromMarkdown` の正規化表現を強化し、予期せぬスペースや極端な長文に対する耐性を向上。
- **エラーメッセージの親切化**: コンテキスト未定義時や認証エラー時、次に何をすべきか（`calendit auth login` を実行するなど）を具体的に提案するように UI を改善。

## テスト・検証内容
- 新しい `src/test_runner.ts` により、全 20 項目以上のテストケースが正常にパスすることを確認。
- LLMクライアントが `calendit --help` から自律的に操作を開始できるための「オンボーディング」体験の自動検証。
