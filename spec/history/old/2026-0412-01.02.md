# バージョン 2026-0412-01.02 変更点

## 概要
カレンダー（フォルダ）自体の管理機能の追加と、破壊的変更に対する安全性担保の仕様策定。

## 変更・追加内容
### 予定（イベント）編集の仕様明確化
- 他アプリ（Google純正、macOSカレンダー等）で作成された予定であっても、サービス共通の `id` を保持することで本ツールから編集可能であることを明記。

### カレンダー管理機能の追加
- イベント操作だけでなく、カレンダー（サブカレンダー）自体の「作成」「名変更」「削除」をサポートする以下のコマンドを追加。
  - `calendit cal list`
  - `calendit cal add <name>`
  - `calendit cal edit <id>`
  - `calendit cal delete <id>`

### 安全性の担保 (Safety Mode) [NEW]
- **Dry Run モード**: `--dry-run` オプションにより、実際の API 実行前に変更内容を確認可能にする。
- **確認プロンプト**: カレンダー削除等の実行前に `Are you sure?` の確認を必須化。
- **保護機能**: プライマリカレンダー（メインのカレンダー）自体の削除を制限。
