# バージョン 2026-0412-01.16

## 概要
テストフレームワークの高度化とマルチプロバイダ（Outlook）サポートの基盤構築、および日付跨ぎ予定の自動処理。

## 変更・追加内容
### Outlook サポートの強化
- **Outlook 認証の完全実装**: `calendit auth login outlook` を実装し、ブラウザベースの対話型ログインによるトークン取得・永続化をサポート。
- **カレンダー管理コマンドの共通化**: `cal list`, `cal add`, `cal delete` にて `--set <name>` オプションをサポートし、プロバイダを問わず同じ使い勝手でカレンダーを操作可能に。

### テストインフラストラクチャの改善
- **高度なテストランナー (`src/test_runner.ts`)**: 
  - `CALENDIT_TEST_CONTEXT` 環境変数による、特定プロバイダに限定した機能テストの実行をサポート。
  - テスト実行時の背景（コンテキスト/プロバイダ）を明示的に出力し、検証結果の透明性を向上。
  - 既存コマンドへの `--set` オプションの自動付与ロジックを最適化。
- **Provider Compliance Suite (互換性テスト群)**: プロバイダに関わらず共通でパスすべきテストケースを定義。

### ユーザビリティとロジックの改善
- **日付跨ぎ予定の自動処理**: 
  - `22:00 - 02:00` のような、終了時刻が開始時刻より前の入力を「翌日の午前中」と自動識別するロジックを `add` コマンドおよび Markdown パーサーに導入。
  - これにより、夜間の予定登録時のエラーを解消。
- **エラーメッセージの親切化**: コンテキストが見つからない場合に、作成コマンドを提案するメッセージを表示するように変更。

## テスト・検証内容
- `npm test` による全機能の回帰テスト。
- `CALENDIT_TEST_CONTEXT` を用いた Google/Outlook 両方のサービスロジックの検証（一部 Dry Run 含む）。
- `TEST_FRESH_INSTALL=true` による、初回インストール時の挙動の自動検証。
