# Version 2026-0416-01.02

## 変更内容
- 品質改善リファクタリングを実施し、デバッグ容易性・堅牢性・高速性・エラーハンドリング・ユーザーフレンドリーな出力を強化。
- CLI の責務を分割し、`index.ts` を薄いエントリポイント化。

## 修正詳細
- `src/core/errors.ts` を新規追加し、`CalendarError` / `ConfigError` / `AuthError` / `ValidationError` / `ApiError` を導入。
- `src/core/logger.ts` を新規追加し、ログレベル制御 (`debug/info/warn/error`) と `--verbose` 連動を導入。
- `src/core/config.ts` を型安全化し、`zod` による設定ファイル検証と型付きアクセサ (`getGoogleCreds`, `setOutlookCreds` など) を追加。
- `src/core/datetime.ts` を新規追加し、日時パース処理を分離。未知フォーマットは `ValidationError` を返すように変更。
- `src/services/outlook.ts` の `pea` typo を `pca` に修正。Graph API 失敗時に `ApiError` を返すよう改善。タイムゾーンをローカル基準に統一。
- `src/services/google.ts` に API エラーラップ処理を追加し、操作単位で原因が分かるメッセージに改善。
- `src/core/formatter.ts` に `ParseResult` と `zod` バリデーションを導入。Markdown パースの警告収集と strict 運用を可能化。
- `src/core/applier.ts` のバリデーション失敗を `ValidationError` 化し、ユーザー向けヒントを追加。
- `src/commands/` 配下に `auth.ts`, `config.ts`, `query.ts`, `apply.ts`, `add.ts`, `cal.ts`, `shared.ts` を新規追加してコマンドを分割。
- `src/index.ts` をコマンド登録と中央エラーハンドリングに専念させる構成へ変更。
- `src/test_runner.ts` を改善し、独立テストケースを `Promise.allSettled` で並列実行して高速化。
