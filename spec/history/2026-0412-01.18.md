# Version 2026-0412-01.18

## 変更内容
- **日付指定のローカル時刻対応 (UTCバグ修正)**:
  - `parseDateTime` 関数において、`YYYY-MM-DD` 形式の入力が UTC 00:00:00 と解釈（JSTでは午前9時開始）されてしまう問題を修正。
  - ハイフン区切りの日付入力を明示的に分割し、ローカル時刻としてインスタンス化するように変更しました。これにより、`--start 2026-04-13` のように指定した際、日本の午前0時からの予定が正しく含まれるようになります。

## 修正詳細
- `src/index.ts` の `parseDateTime` 内で、正規表現 `/^\d{4}-\d{2}-\d{2}$/` を用いた判定処理を追加。

## 検証結果
- `node ./bin/cli.js query --calendar primary --start 2026-04-13` を実行し、朝8時台の予定（朝礼：飯澤など）が正しく出力されることを確認。
